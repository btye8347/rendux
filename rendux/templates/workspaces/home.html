Welcome to RendUX

A config-driven UI shell runtime. Views are declared in YAML, rendered by the runtime, and extended with injectable application logic.

Config-driven

Views, routes, and shell structure are declared in config/views.yaml. No code required to add a new view.

HTMX-native

Navigation swaps only the workspace content. Full pages served on direct load; partials served on HTMX requests.

Injectable logic

Application routes and data endpoints plug into the shell. The framework handles rendering; your code handles the data.
